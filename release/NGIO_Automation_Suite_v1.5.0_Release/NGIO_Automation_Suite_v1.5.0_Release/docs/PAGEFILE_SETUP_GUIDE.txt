        🚨 CRITICAL: Windows Pagefile Setup for Overnight Generation 🚨
================================================================================

ISSUE: Skyrim Can Crash During Overnight Generation (Especially <16GB RAM)

During grass cache generation, Skyrim can consume ALL available RAM (12-20GB 
with heavy mod lists). Without proper Windows pagefile (virtual memory), your 
system will run out of memory and crash.

This is ESPECIALLY critical for:
• Systems with 16GB RAM or less
• Heavy mod lists (500+ mods)
• "All 4 Seasons" overnight mode
• Extended settings (SuperDenseGrass, ExtendGrassCount)

================================================================================
                        SOLUTION: Configure 20GB Pagefile
================================================================================

⏱️  Takes 2 minutes | ✅ One-time setup | 🛡️ Prevents crashes forever

STEP-BY-STEP INSTRUCTIONS:

1️⃣  Open Windows System Properties:
   
   Method A (Easiest):
   • Press Windows key, type "View advanced system settings"
   • Click the result
   
   Method B:
   • Right-click "This PC" or "Computer" on desktop/File Explorer
   • Click "Properties"
   • Click "Advanced system settings" on left side

2️⃣  Open Performance Settings:
   
   • In "Advanced" tab (already selected)
   • Under "Performance" section
   • Click "Settings..." button

3️⃣  Configure Virtual Memory:
   
   • Click "Advanced" tab in Performance Options
   • Under "Virtual memory" section
   • Click "Change..." button

4️⃣  Set Custom Pagefile Size:
   
   • UNCHECK "Automatically manage paging file size for all drives"
   • Select your main drive (usually C:)
   • Select "Custom size" radio button
   • Set Initial size (MB): 20480
   • Set Maximum size (MB): 20480
   • Click "Set" button
   • Click "OK"
   • Click "OK" again
   • Click "OK" to close System Properties

5️⃣  Restart Windows:
   
   • Required for pagefile changes to take effect
   • After restart, pagefile is configured!

================================================================================
                        TECHNICAL EXPLANATION
================================================================================

What is Pagefile?
• Virtual memory on your disk that acts as "overflow" RAM
• When RAM fills up, Windows moves less-used data to pagefile
• Allows programs to use more memory than physical RAM available

Why 20GB?
• Skyrim during generation: 8-12GB
• Heavy mod lists: +2-6GB
• Windows OS: 2-4GB
• Other programs: 2-4GB
• Safety buffer: 2-4GB
• Total: 16-30GB potential usage

With 16GB RAM:
• ~14GB usable (2GB reserved for Windows)
• 20GB pagefile = 34GB total virtual memory
• Enough for worst-case grass generation

With 32GB RAM:
• Pagefile less critical but still recommended
• Provides safety buffer for extreme mod lists

Performance Impact:
• Minimal! Pagefile on SSD is fast
• Only used when RAM actually fills up
• Better than crashing and losing hours of progress!

Cost:
• 20GB disk space (one-time allocation)
• Worth it to prevent crashes!

================================================================================
                            TROUBLESHOOTING
================================================================================

Q: "I set 20GB but validation still shows 8GB?"
A: Restart Windows! Pagefile changes require restart.

Q: "I don't have 20GB free space?"
A: Minimum 10GB will help. Clean up disk space if possible.
   Or use different drive (D:, E:) for pagefile.

Q: "Will this slow down my computer?"
A: No! Modern SSDs make pagefile very fast. Only used when needed.

Q: "Can I set it back after generation?"
A: Yes, but why? It's a one-time setup that prevents future crashes.

Q: "I have 32GB RAM, do I need this?"
A: Probably not for single seasons, but recommended for "All 4 Seasons" 
   overnight mode with heavy mod lists.

Q: "My generation crashed anyway?"
A: Check:
   • Task Manager during generation - is RAM at 100%?
   • Is pagefile actually configured? (Restart required!)
   • Do you have other programs using RAM? (Close them)
   • Is your disk full? (Pagefile needs space)
