⚠️ 🚨 CRITICAL: OVERNIGHT MODE SYSTEM REQUIREMENTS 🚨 ⚠️

IMPORTANT FOR USERS WITH <16GB RAM:

Skyrim can consume ALL available RAM during grass generation!
Your system can CRASH during overnight generation without proper pagefile!

REQUIRED SETUP (Takes 2 minutes, prevents crashes):

1. Open Windows System Properties:
   • Right-click "This PC" → Properties → Advanced system settings
   • Or search "View advanced system settings"

2. Configure Virtual Memory:
   • Click "Settings" under Performance
   • Go to "Advanced" tab
   • Click "Change..." under Virtual Memory
   • UNCHECK "Automatically manage paging file size"
   • Select your drive (usually C:)
   • Select "Custom size"
   • Set BOTH Initial and Maximum to: 20480 MB (20GB)
   • Click "Set" then "OK"
   • Restart Windows

3. Why This Matters:
   • Skyrim + Heavy mod lists can use 12-20GB during generation
   • With 16GB RAM, you have ~14GB usable (2GB for Windows)
   • Pagefile acts as "overflow" RAM on your disk
   • 20GB pagefile = guaranteed crash prevention
   • One-time setup, works for all future generations!

RECOMMENDED: 32GB RAM users can skip this, but 20GB pagefile still helps!

Tool automatically checks your pagefile and warns if too small.

5. 🆘 TROUBLESHOOTING (Common Issues & Solutions)
================================================================================

❌ "Failed to find bundled Python" or Python-related errors
Problem: Corrupted or incomplete extraction
Solution:
1. Re-download the bundled zip file
2. Extract to a new location using Windows built-in extractor
3. Make sure the venv/ folder exists in the extracted directory
4. Run start_ngio_automation.bat from the extracted folder

❌ "SKSE loader not found" or "Skyrim process not found"
Problem: SKSE64 not installed properly
Solution:
1. Download SKSE64 from https://skse.silverlock.org/
2. Extract ALL files to your Skyrim directory (where SkyrimSE.exe is)
3. Verify skse64_loader.exe exists in Skyrim folder
4. Test: Launch Skyrim using skse64_loader.exe manually

❌ "No grass generation" or "PrecacheGrass.txt not created"
Problem: NGIO mod not working
Solution:
1. Ensure NGIO is ✅ ENABLED in your mod manager
2. Check NGIO is not being overridden by other mods
3. Verify SKSE64 is working (see above)
4. Try launching Skyrim manually first to test

❌ "Skyrim won't launch" or "Process terminated immediately"
Problem: Skyrim installation or configuration issue
Solution:
1. Verify Skyrim path is correct (where SkyrimSE.exe is located)
2. Launch Skyrim manually first to ensure it works
3. Check for conflicting mods or mod manager issues
4. Try running as Administrator

❌ "Permission denied" or "Access denied" errors
Problem: Windows permissions blocking file operations
Solution:
1. Run the tool as Administrator (right-click → Run as administrator)
2. Check your antivirus isn't blocking the tool
3. Ensure output directory has write permissions
4. Try using a different output directory (like Desktop)

❌ "Archive not created" or "Files not found"
Problem: File processing or archive creation failed
Solution:
1. Check output directory has enough free space (5GB+)
2. Verify no other programs are using the Skyrim directory
3. Check antivirus isn't quarantining files
4. Try running as Administrator

❌ Tool crashes or hangs during generation
Problem: Various possible causes
Solution:
1. Check system has enough RAM (8GB+ recommended)
2. Close unnecessary programs to free memory
3. Try increasing timeout values in configuration
4. Check logs for specific error messages

❌ "Seasons of Skyrim config not found"
Problem: Seasonal mod not installed or configured
Solution:
1. Install Seasons of Skyrim from Nexus Mods
2. Configure it in your mod manager
3. Or select "Non-seasonal mode" in the tool

🎯 STILL HAVING ISSUES?
1. Check the logs - Tool creates detailed logs for troubleshooting
2. Try non-seasonal mode - Simpler setup, fewer dependencies
3. Test with minimal mods - Disable other mods temporarily
4. Report bugs - Include your Skyrim version and error messages

📋 BEFORE ASKING FOR HELP, PROVIDE:
• Skyrim version (SE/AE/VR)
• NGIO mod version
• Whether SKSE64 works manually
• Full error message or log file
• Your mod list (if relevant)
• Confirm you're using the bundled version

7. ❓ FAQ
================================================================================

Q: Do I need to disable NGIO after using this tool?
A: No! Keep NGIO enabled in case you want to generate more seasons later. 
   The generated archives work alongside NGIO.

Q: Can I use this without seasonal mods?
A: Yes! Select "Non-seasonal mode" during setup. This generates a universal 
   grass cache that works without seasonal dependencies.

Q: How long does generation take?
A: 20-60 minutes per season, depending on your mod list complexity. Heavy 
   grass mods and large worldspaces take longer.

Q: What if my generation gets interrupted?
A: The tool preserves progress in PrecacheGrass.txt. When you restart, it 
   will ask if you want to resume or start fresh.

Q: Can I generate multiple seasons at once?
A: Yes! Use --season all in CLI mode. However, single-season workflow is 
   recommended for optimal disk space usage. Run it multiple times for 
   different seasons if disk space is limited.

Q: Does this work with other grass mods?
A: Yes! Any mod that adds grass objects will be included in the cache. The 
   tool works with the NGIO plugin, not specific grass mods.

Q: What's the difference between crash timeout and no-progress timeout?
A: Crash timeout (5min) detects when Skyrim process stops running. 
   No-progress timeout (15min) detects when generation stalls but process 
   is still alive.

Q: Can I use this on a different computer?
A: Yes! The bundled version works on any Windows PC. 
   Just copy the extracted folder and run it - no installation needed.

Q: What are the .sha256 files for? (NEW in v1.2.0)
A: These are checksum files for verifying archive integrity. They ensure
   your download wasn't corrupted. Use PowerShell's Get-FileHash to verify.

Q: Can I schedule generation to run overnight? (NEW in v1.2.0)
A: Yes! Use Windows Task Scheduler with the --unattended flag:
   start_ngio_automation.bat --season winter --unattended

Q: How do I disable notifications? (NEW in v1.2.0)
A: Notifications are optional. If win10toast is not installed, they're
   automatically disabled. You can also modify the notification settings
   in the code if needed.

Q: What does --dry-run do? (NEW in v1.2.0)
A: It simulates the entire workflow without actually launching Skyrim or
   making any changes. Perfect for testing your configuration safely.
